A legible instrument

Method

01

Observe, never run

The scanner reads bounded text files. It does not import, build, install, evaluate, or execute project code.

02

Arrange atomic evidence

A Cupboard piece is a donor observation connected to one host direction. Multi-project arrangements preserve every path, line, rule, and caution.

03

Steer explicitly

Breadth, novelty, compatibility, risk, focus, donor preferences, locks, and exclusions change deterministic ranking. They never invent evidence.

04

Keep claims bounded

Scores and suggestions are heuristics grounded in observed Evidence. They never claim project quality, buildability, API equivalence, license compatibility, or correctness.

05

Project, then copy

The Mashup map, canonical Affinity projection, Conceptual Mashup, and Recombination Brief are browser-local projections. They do not alter the Cabinet Snapshot, call a model, execute code, or persist workspace state.
